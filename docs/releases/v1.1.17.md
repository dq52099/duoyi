# v1.1.17

本次发布继续收口线上服务、日历万年历、今日任务和发布检测问题。

## 重点修复

- 线上 P0 服务链路复测通过：个人资料保存、头像上传、密码修改、邮箱测试、用户与用户组、时光币调整、专注自习室接口均走真实后端。
- 万年历改用 `lunar` 数据源，补齐农历、干支、节气、宜忌、胎神、彭祖、五行、星宿、冲煞和时辰吉凶展示，不接入天气。
- 今日待办和今日提醒保留真实数据展示，完成、逾期、正常事项使用不同样式，图标与任务名同一行。
- 今日打卡任务图标回退逻辑与新建任务图标保持一致，不再用勾号代替习惯图标。
- 主待办列表和看板左滑不再重复展示“详情”，点击卡片负责查看详情，左滑只保留完成/恢复和删除。
- 默认主题二级实心按钮会自动调整到可读对比度，跳转页 AppBar 标题字号与主页保持一致。
- 月视图任务标记在较小日期格中也显示任务数量，搜索纪念日可跳转并回显目标。
- 更新检测版本升至 1.1.17，后端配置与客户端 AppVersion 同步。
- Release 签名链路不变，继续使用正式 keystore，不允许 debug 签名发布。

## 验证

- `/home/ubuntu/flutter/bin/flutter analyze`
- 受影响 Flutter 测试集：万年历、日历月视图、搜索跳转、今日待办/提醒、习惯图标、更新策略
- 后端 P0 契约测试
- 线上真实 E2E：用户资料、头像、密码、用户组、时光币、邮箱、自习室
