# v1.1.18

本次发布重点收口数据同步、主题时光币一致性和专注页卡顿问题。

## 重点修复

- 同步机制改为合并通知与排队同步，避免登录、回写和前台刷新期间触发并发同步。
- 云同步回写后按命中集合分批 reload，减少一次同步导致多个 Provider 同时重建。
- 启动延迟加载和云端回写 reload 不再误标记为本地脏数据，避免无意义二次同步。
- 账号刷新增加 requestId、token、userId 和 mutation 序号校验，旧响应不会覆盖刚完成的主题或时光币状态。
- 主题商店和时光币快照按 updatedAt 拒绝旧数据，修复切换主题后自动回切、消费后余额不刷新的风险。
- 专注页自习室、排行榜和白噪音信息改为局部订阅，减少排行榜刷新拖慢主计时页。
- 保持 release 签名链路不变，继续使用正式 keystore，不改签名配置。
