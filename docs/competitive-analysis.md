## 2. 多仪现有能力评估

### 2.1 核心优势

多仪是一款**功能高度集成**的个人效率工具，在单一应用内聚合了竞品需要 2-3 个独立 app 才能覆盖的场景：

| 能力域 | 实现深度 | 评价 |
|---|---|---|
| 待办管理 | 四象限 + 5级优先级 + 子任务 + 重复规则 + 自动顺延 + 标签分组 | **较完整**，核心数据模型健全 |
| 习惯追踪 | 正/负向 + 计量单位 + 热力图 + 连续天数 + 周目标 + 分类标签 | **业内少有**的正负向双模型和热力图 |
| 目标管理 | 6 分类 + 固定/随机调度 + 推荐目标库 + 节假日跳过 | **独占优势**，竞品均无独立目标模块 |
| 专注模式 | 番茄钟(25/5/15 可配) + 9种白噪音 + FocusLink 任务关联 | 基本完整，与任务联动是亮点 |
| 日历 | 月/周/日 三视图 + 农历/节气 + 多数据源聚合 | 够用，但视图数量少于滴答清单 |
| 日记 | 心情记录 + 心情热力图 + 连续天数统计 | 差异化功能，竞品普遍缺失 |
| 笔记 | 轻量便签 | 最小可用 |
| 课程表 | 周课程格 | 对学生群体有用 |
| 倒数日/纪念日 | 独立模块 + 提醒 | 竞品多需独立 app |
| 老黄历 | 农历转换 + 节气 + 宜忌 | 中国用户文化刚需 |
| 提醒系统 | 双通道(push/alarm) + ReminderPlan 多次提醒 | 架构先进，分离通知和闹钟 |
| 成就系统 | 多维度徽章 + 进度条 + AchievementEngine | 有体系，但缺社交激励 |
| 统计 | 任务/专注/习惯/日记 + 时间审计 + 周/月/年 | 基本完备 |
| 数据同步 | 自建后端 + 云同步 + JSON 备份 + ICS 导出 | 数据主权在用户手中 |
| 共享协作 | Workspace 模型 + 邀请码 + 角色权限(owner/editor/viewer) | **已有框架**，但功能深度不足 |
| AI | 任务分解 + 周报复盘 + 服务端代理 | 已落地但场景有限 |
| 跨平台 | Flutter（Android/iOS/macOS/Windows/Web） | 一套代码全平台覆盖 |

### 2.2 技术栈评估

- **框架**: Flutter + Provider 状态管理
- **本地持久化**: SharedPreferences（轻量但面临大数据量瓶颈）
- **后端**: 自建 API（Go/Node 推测），支持认证、同步、AI 代理
- **通知**: 双通道架构 (duoyi_general / duoyi_alarm)，ReminderScheduler 统一调度
- **音频**: audioplayers 包，ReleaseMode.loop 无缝循环
- **图表**: fl_chart
- **小组件**: home_widget 包，仅 Android（2种布局）

## 3. 核心差距分析（按功能维度）

### 3.1 任务管理

#### 已有
- 四象限分类（`EisenhowerQuadrant` 四值枚举）
- 5 级优先级（none/low/medium/high/urgent）
- 子任务 (`Subtask` 模型) 含自动驱动父任务完成态
- 重复规则 (`RecurrenceRule`)
- 标签、分组 (`listGroupId/listGroupName`)
- 自动顺延 (`PostponeRecord` + `DailyRollover`)
- 快速添加 (`QuickCaptureFab`)
- 排序 (sortOrder + 优先级倒序 + 创建时间)
- 工作区隔离 (`workspaceId`)
- 完成归档策略 (`CompletionVisibilityPolicy`)

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **看板视图** | 滴答清单、Todoist 均支持 | 项目管理场景缺失，用户流失高危 | **P1** |
| **自然语言日期解析** | 滴答清单"下周三提醒我开会"、Todoist "every mon at 9am" | 录入效率差距巨大，是核心体验分水岭 | **P1** |
| **筛选器/自定义视图** | Todoist 的 Filters、滴答清单的智能清单 | 任务多时缺乏高效检索手段 | **P2** |
| **批量操作** | 滴答清单、Todoist 的多选拖拽 | 整理成本高 | **P2** |
| **任务描述富文本** | 滴答清单 Markdown、Todoist 评论区 | 复杂任务描述受限 | **P3** |
| **任务排序拖拽** | 滴答清单、Todoist 拖拽排序 | 手动排序不够直觉 | **P2** |

### 3.2 日历与日程

#### 已有
- 三视图：`CalendarMonthGrid`（月）、`CalendarWeekStrip`（周）、`CalendarDayAgenda`（日）
- 数据聚合：`CalendarAggregator` 整合 Todo/Habit/Pomodoro/Anniversary/Course/Diary/Countdown/Goal
- 农历 + 节气 + 节日 (`LunarCalendar`)
- 快速添加任务到指定日期
- ICS 导出 (`IcsExporter`)

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **时间线视图（Timeline）** | 滴答清单的时间线、番茄TODO 的时间轴 | 无法按时间块可视化一天 | **P2** |
| **三日视图** | 滴答清单 | 短期规划场景缺失 | **P3** |
| **第三方日历订阅（CalDAV/Google/Outlook）** | 滴答清单、Todoist、Microsoft To Do | 严重限制：无法与用户现有日程整合 | **P1** |
| **日程事件拖拽调整** | 滴答清单日历拖拽 | 日程调整效率低 | **P3** |
| **全天事件/多日事件** | 主流日历 App | 出差、假期等场景缺失 | **P2** |

### 3.3 提醒系统

#### 已有
- 双通道架构：`duoyi_general`（通知推送） + `duoyi_alarm`（闹钟级全屏高优先级）
- `ReminderConfig`（kind: push/alarm + hour/minute）
- `ReminderPlan`（多次提醒计划）
- `ReminderScheduler` 统一调度器
- 通知健康检查 (`PermissionHealthService`)

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **位置提醒** | 滴答清单"到达公司时提醒"、Todoist 地理围栏 | 场景性需求，上下班/购物时提醒 | **P3** |
| **提醒偏移量模板** | 滴答清单"提前15分钟/1小时/1天" | 当前需手动设置时间，配置成本高 | **P2** |
| **邮件提醒** | Todoist 的邮件提醒通道 | 桌面端弱通知场景 | **P4** |

### 3.4 专注模式

#### 已有
- 番茄钟：可配时长（默认 25/5/15 分钟）+ 4 轮长休息
- 白噪音：9 种（rain, forest, cafe, waves, brown_noise, night_rain, fan, pink_noise, deep_stream）
- `FocusSoundService`：单例播放、loop 循环、fadeIn/fadeOut
- `FocusLink`：任务与专注预设绑定
- 自动开始休息/专注可配
- 休息阶段白噪音可配
- 专注历史记录 (`PomodoroSession`)
- 今日专注统计（时长 + 次数）

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **白噪音混合/叠加播放** | 番茄TODO 支持多音轨混合（雨+雷+钢琴）| 用户期望高，体验差距明显 | **P1** |
| **白噪音数量** | 番茄TODO 30+ 种（含自然/城市/乐器等分类）| 9种偏少 | **P2** |
| **锁屏/屏蔽干扰 App** | 番茄TODO "学霸模式"、Forest 核心功能 | 无法强制专注，学生/自律场景缺失 | **P2** |
| **自习室/社交专注** | 番茄TODO 自习室、Forest 多人种树 | 社交激励完全缺失 | **P2** |
| **正计时模式** | 滴答清单持续时间 | 非番茄节奏的自由计时 | **P3** |
| **专注报告导出** | 番茄TODO 详细报告 | 无法生成分享图 | **P3** |

### 3.5 习惯追踪

#### 已有
- 正/负向双模型 (`HabitKind.positive / .negative`)
- 计量单位 (`unit`: 次/杯/分钟)
- 热力图 (`HabitHeatmap`)
- 连续天数 / 最佳连续 (`currentStreak / bestStreak`)
- 活跃星期 (`activeWeekdays`)
- 每周目标 (`weeklyTarget`)
- 分类 + 标签
- 开始/结束日期
- 提醒 (`remind + remindHour/remindMinute`)

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **习惯分组/文件夹** | 滴答清单习惯分组 | 习惯多时管理混乱 | **P3** |
| **习惯模板库** | 代码中有 `habit_templates.dart` 但需验证覆盖度 | 降低新用户创建门槛 | **P3** |
| **习惯完成动画/反馈** | 番茄TODO 打卡音效 | 感官反馈弱 | **P4** |
| **习惯数据图表（趋势线）** | 滴答清单习惯统计 | 长期趋势不直观 | **P3** |

> 注：多仪的习惯模块**总体领先**大部分竞品（正/负向 + 热力图 + 计量单位的组合在竞品中罕见），差距较小。

### 3.6 统计分析

#### 已有
- 周/月/年三维度切换 (`_Range.week / .month / .year`)
- 任务完成数、专注时长/次数、日记条数、习惯完成天数
- fl_chart 图表
- 时间审计 (`TimeAuditScreen` + `TimeAuditProvider`)
- 时间分类聚合 (`secondsByCategory`)

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **生产力评分/综合指数** | Todoist Karma、番茄TODO 学习力 | 缺乏一目了然的"效率指标" | **P2** |
| **对比分析（同比/环比）** | 番茄TODO "与上周对比" | 无法感知进步 | **P2** |
| **统计报告导出/分享** | 番茄TODO 分享图片、滴答清单 PDF | 无法分享给他人激励 | **P3** |

| **热力图（GitHub 风格）** | 番茄TODO 年度日历 | 全局时间分布不直观 | **P3** |

#### 缺失 — 与竞品差距

| 缺失功能 | 竞品参照 | 影响评估 | 优先级 |
|---|---|---|---|
| **任务指派（Assignee）** | 滴答清单、Todoist、Microsoft To Do | 团队场景无法分工 | **P1** |
| **任务评论/讨论** | 滴答清单、Todoist 评论区 | 无法在任务上沟通 | **P2** |
| **活动流/操作日志** | 滴答清单共享变更记录 | 无法追踪"谁改了什么" | **P2** |
| **@提及通知** | Todoist | 无法定向通知协作者 | **P3** |

| **专注计时 Widget** | 番茄TODO | 无法在桌面看到倒计时 | **P3** |

#### 已有

## 4. 多仪差异化优势

以下是多仪相对于所有对标竞品的**独占或领先优势**，这些是产品护城河，应当持续强化：

### 4.1 全场景一体化（最核心优势）

多仪在**单一 App** 内集成了：待办 + 习惯 + 目标 + 日历 + 番茄钟 + 日记 + 笔记 + 课程表 + 倒数日 + 纪念日 + 老黄历。

- 滴答清单覆盖 ~70%（缺日记/笔记/目标/倒数日/纪念日/黄历）
- Todoist 覆盖 ~40%（纯任务管理 + 日历）
- 番茄TODO 覆盖 ~50%（缺日历/目标/笔记/纪念日/黄历）
- Forest 覆盖 ~15%（仅专注）
- Microsoft To Do 覆盖 ~30%（纯任务 + 日程）

**用户价值**：避免多 App 切换、数据孤岛，所有生活/学习/工作数据集中管理。

### 4.2 独立目标管理系统（独占）

`GoalItem` 模型 + 6 分类 + 固定/随机调度 + 推荐目标库 + 节假日跳过——这是**所有 5 个竞品都没有的功能**。

- 目标不同于任务：任务是"做什么"，目标是"成为什么"
- 调度策略（随机派发 + 最小间隔 + 周/月上限）支持"不定期推动自己"的场景
- 推荐目标库降低了"不知道设什么目标"的冷启动门槛

### 4.3 正/负向习惯双模型（领先）

`HabitKind.positive` (养成) + `HabitKind.negative` (戒除) 在竞品中罕见：
- 滴答清单仅正向
- 番茄TODO 仅正向
- 负向习惯（戒烟/戒糖/减少手机使用）是真实用户需求

### 4.4 习惯热力图（领先）

`HabitHeatmap` 类似 GitHub 贡献图，在效率 App 中独一无二：
- 滴答清单用柱状图
- 番茄TODO 用日历格
- 热力图在长时间维度下更直观

### 4.5 中国文化特色功能集（差异化）

- **农历/节气**：`LunarCalendar.fromSolar()` + `solarTerm()` + 节日检测
- **老黄历**：`AlmanacScreen` 宜忌查询
- **纪念日**：农历循环纪念（竞品多只支持公历）
- **倒数日**：独立模块，非简单的提前提醒

这些功能在国际化竞品（Todoist、Microsoft To Do、Forest）中完全缺失。

### 4.6 双通道提醒架构（技术领先）

`duoyi_general`（普通通知） + `duoyi_alarm`（闹钟级全屏震动），由 `ReminderScheduler` 统一调度——大多数竞品只有单一通知通道，多仪的分离架构更符合 Android 通知规范。

### 4.7 数据主权/自部署模式

自建后端 + 全量 JSON 导出 + ICS 导出——用户完全掌控数据。对隐私敏感用户（特别是中国市场对数据安全的关注）有天然吸引力。

### 4.8 AI 周报（独占）

`weeklyReview()` 基于本周真实数据生成个性化复盘——目前无竞品有此功能（Todoist 的 Karma 是静态算分，不会生成文字建议）。

## 5. 优先补齐建议

基于影响面、实现复杂度、竞品差距综合评估，建议分三阶段推进：

### 第一阶段：核心体验补齐（P0-P1，预计 4-6 周）

解决"用户试用 5 分钟内就能感知到的差距"。

| # | 功能 | 理由 | 预估工作量 |
|---|---|---|---|
| 1 | **自然语言日期解析** | 录入效率是任务管理 App 的生死线。滴答清单和 Todoist 的核心竞争力都建立于此。可基于现有 AI 服务端做中文日期抽取，或本地集成 jieba + 规则引擎。 | 2-3 周 |
| 2 | **iOS Widget** | iOS 用户占比高，无 Widget 等于放弃桌面入口。Flutter 的 home_widget 包已支持 iOS，主要工作在 Swift 侧。 | 1-2 周 |
| 3 | **第三方日历订阅** | 用户已有 Google/Outlook 日程——无法整合 = 无法替代现有工具。实现 CalDAV 只读订阅（解析 ICS URL）是最小成本方案。 | 2 周 |
| 4 | **任务指派** | 共享空间已有 Workspace + 角色体系，`TodoItem` 加 `assigneeId` 字段即可打通任务分配。 | 1 周 |

### 第二阶段：竞争力增强（P2，预计 6-8 周）

解决"用户深度使用后感知到的不足"。

| # | 功能 | 理由 | 预估工作量 |
|---|---|---|---|
| 5 | **看板视图** | 项目管理场景刚需，实现基于 `listGroupId` 分列 + 拖拽移动。 | 2 周 |
| 6 | **白噪音混合播放** | 番茄TODO 的核心差异点，需改造 `FocusSoundService` 为多 `AudioPlayer` 实例 + 音量滑块。 | 1-2 周 |
| 7 | **生产力评分** | 综合待办完成率 + 专注时长 + 习惯连续天数，计算日/周"效率指数"，显示在 TodayScreen。 | 1 周 |
| 8 | **增量同步** | 当前全量同步不可扩展，需实现 `updatedAt` 比较 + 字段级 merge + 冲突策略。 | 3 周 |
| 9 | **更多 Widget 尺寸+类型** | 快捷添加 Widget、习惯打卡 Widget、专注倒计时 Widget。 | 2 周 |
| 10 | **智能日程建议** | 类 Microsoft To Do "My Day"：根据截止日、优先级、历史完成率，每日推荐 5-7 条任务。 | 1-2 周 |
| 11 | **筛选器/自定义视图** | 按标签/优先级/日期范围/象限组合筛选，保存为"智能清单"。 | 2 周 |
| 12 | **统计对比分析** | 同比/环比 + 趋势折线图 + "比上周提升 X%"。 | 1 周 |
| 13 | **提醒偏移量模板** | 提前 5 分钟/15 分钟/1 小时/1 天预设选项。 | 0.5 周 |

### 第三阶段：生态与增长（P3-P4，按需推进）

解决"用户留存和传播"。

| # | 功能 | 理由 | 预估工作量 |
|---|---|---|---|
| 14 | 社交专注/自习室 | 番茄TODO 和 Forest 的增长引擎 | 4-6 周 |
| 15 | 虚拟货币/奖励商店 | 游戏化留存 | 3 周 |
| 16 | 排行榜 | 社交竞争 | 2 周 |
| 17 | 统计报告导出/分享图 | 社交传播 | 1 周 |
| 18 | 富文本笔记（Markdown） | 笔记场景深化 | 2 周 |
| 19 | 图片/附件支持 | 任务和笔记附件 | 2-3 周 |
| 20 | 第三方集成（API 开放） | 生态扩展 | 持续 |
| 21 | 锁屏/屏蔽干扰 App | Android 需要辅助功能权限，iOS 受限 | 2-3 周 |
| 22 | 从竞品导入数据 | 降低迁移成本 | 2 周 |
| 23 | CSV/Excel 导出 | 数据可移植性 | 1 周 |
| 24 | 备份到云盘 | Google Drive/iCloud 备份 | 2 周 |

### 投入产出比总结

```
投入最小、效果最大（推荐立即做）:
  → 自然语言日期解析、iOS Widget、提醒偏移量模板

投入中等、差异化显著（推荐近期做）:
  → 白噪音混合、生产力评分、看板视图

投入大、生态价值高（推荐规划做）:
  → 第三方日历订阅、增量同步、社交专注
```
